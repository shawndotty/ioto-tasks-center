# 根据模板批量创建任务的方案

## 背景

当前 IOTO Task Center 插件的任务创建流程是**逐个创建**：每次只能创建一个任务文件。对于固定 SOP（标准操作流程）等场景，需要批量创建多个步骤性任务，效率较低。

## 现有架构分析

### 任务创建核心流程

```
TaskCreationModal（收集名称/优先级/星标）
    ↓
createTaskFile(options) → 创建空文件 → 应用模板 → 应用属性
    ↓
applyCreatedTaskSettings() → 设置优先级/星标
```

关键函数 `createTaskFile()` 的职责：

1. 按命名规则生成文件名（`buildTaskFileName()`）
2. 解析目标路径（`resolveTaskTargetPath()`）
3. 创建空文件 → 应用模板 → 应用 `Project` 等 frontmatter 属性

### 模板系统现状

支持四种任务类型，每种有独立模板配置：

| 类型     | 文件名模式              | 模板来源 |
| -------- | ----------------------- | -------- |
| `normal` | `{名称}.md`             | 文件模板 |
| `date`   | `{项目}-{日期}.md`      | 文件模板 |
| `topic`  | `{项目}-主题-{名称}.md` | 内联模板 |
| `plan`   | `{项目}-计划-{名称}.md` | 文件模板 |

模板来源支持两种模式：`file`（引用模板文件）和 `inline`（内联内容）。

## 方案设计

### 核心思路

在设置中新增**批量任务模板**（Batch Task Templates），每个模板由一个 Markdown 列表定义，列表的每一项对应一个待创建的任务。用户触发批量创建时，选择一个模板，输入可选前缀，插件按模板逐个创建任务文件并自动建立父子关系。

### 1. 批量模板的数据结构

在 `settings.ts` 中新增类型定义：

```typescript
// 单条批量任务模板
interface BatchTaskTemplate {
	id: string; // 唯一标识，用于设置中的引用
	name: string; // 模板显示名称，如 "新员工入职 SOP"
	taskType: TaskCreationType; // 创建的任务类型（normal/date/topic/plan）
	listContent: string; // Markdown 列表内容（模板本体）
}

// 批量模板配置
interface BatchTemplateConfig {
	enabled: boolean; // 是否启用批量模板功能
	templates: BatchTaskTemplate[]; // 模板列表
}
```

`listContent` 是一个 Markdown 列表，格式示例：

```markdown
- 签订劳动合同
- 提交入职材料
    - 身份证复印件
    - 学历证书复印件
    - 离职证明
- 开通企业邮箱
- 分配工位和设备
- 安排入职培训
    - 公司文化培训
    - 部门业务培训
    - 系统权限培训
```

### 2. 列表解析规则

定义一个 `parseBatchList(content: string)` 函数，将 Markdown 列表解析为结构化的任务树：

```typescript
interface BatchTaskItem {
	name: string; // 任务名称
	level: number; // 缩进层级（0 = 顶级）
	parentIndex: number | null; // 父任务在数组中的索引
}

function parseBatchList(content: string): BatchTaskItem[];
```

解析规则：

- 以 `- ` 开头的行视为一个任务条目
- 缩进层级通过前导空格数判断（2 空格 = 1 级）
- 无缩进的任务（层级 0）为顶级任务
- 有缩进的任务（层级 ≥ 1）的父任务 = 向上最近的层级更小的任务
- 忽略空行和非列表行
- 去除任务名称首尾空格

### 3. 前缀处理

批量创建时，用户可输入前缀。最终的任务名称规则：

```
{前缀}{原任务名称}
```

示例：前缀 = "入职-202606-"，则"签订劳动合同" → "入职-202606-签订劳动合同"

**目的**：

1. 避免同项目下同名笔记冲突
2. 按批次组织任务

前缀存储在每次批量创建的调用参数中，不持久化到设置。

### 4. 任务创建流程

新增 `handleBatchCreateTask()` 函数（在 `iotoTasksCenterView.ts` 中注册）：

```
1. 弹出批量模板选择弹窗（显示所有已配置的模板名称）
2. 弹出前缀输入弹窗（可选，用户可留空）
3. 弹出确认弹窗（展示即将创建的任务列表预览）
4. 确认后，逐个调用 createTaskFile()：
   a. 文件名 = buildTaskFileName({ taskType, customName: 前缀 + 原名 })
   b. 目标路径 = resolveTaskTargetPath()
   c. 应用模板
   d. 应用 Project 属性（继承自当前选中的项目）
   e. 如果是子任务（level ≥ 1），调用 assignUpTaskToFile() 设置 UpTask 属性
5. 完成后显示结果通知
```

### 6. 触发方式

批量创建通过以下方式触发：

1. **命令面板**：新增命令 `批量创建任务（从模板）`
2. **任务中心右键菜单**：在项目上右键，新增菜单项 "批量创建任务"
3. **工具栏按钮**：在任务中心视图的工具栏区域新增批量创建图标按钮

触发后弹出模板选择 → 前缀输入 → 确认 → 执行。
