## 需求背景

在 `IOTO Task Center-计划-V214` 中已经实现了通过设置列表模板来批量创建任务文件的功能。但目前的批量任务模板可以被任何项目全部读取，当用户创建的批量模版过多时，每次选择起来会比较麻烦。

**核心需求**：给每个批量任务的模板提供一个所属项目的属性。这个属性是一个数组值：

- 如果值为空，模板可以被所有项目使用
- 如果属性值中有对应的项目名称，该模板只能被指定的项目使用

```typescript
function areBatchTemplatesEqual(
	left: BatchTaskTemplate,
	right: BatchTaskTemplate,
): boolean {
	return (
		left.id === right.id &&
		left.name === right.name &&
		areLevelTaskTypesEqual(left.levelTaskTypes, right.levelTaskTypes) &&
		left.listContent === right.listContent &&
		areProjectsEqual(left.projects, right.projects)
	);
}

function areProjectsEqual(left: string[], right: string[]): boolean {
	if (left.length !== right.length) {
		return false;
	}
	// 排序后比较，忽略顺序差异
	const sortedLeft = [...left].sort();
	const sortedRight = [...right].sort();
	return sortedLeft.every((project, index) => project === sortedRight[index]);
}
```

```typescript
export class BatchTemplateSelectModal extends Modal {
	private readonly templates: BatchTaskTemplate[];
	private readonly currentProject: string; // 新增
	private filteredTemplates: BatchTaskTemplate[] = []; // 新增：过滤后的模板

	constructor(
		app: Modal['app'],
		templates: BatchTaskTemplate[],
		currentProject: string, // 新增参数
	) {
		super(app);
		this.templates = templates;
		this.currentProject = currentProject;
		this.filteredTemplates = this.filterTemplatesByProject();
	}

	/**
	 * 根据当前项目过滤模板
	 */
	private filterTemplatesByProject(): BatchTaskTemplate[] {
		return this.templates.filter((template) => {
			// 如果 projects 数组为空，则模板可被所有项目使用
			if (!template.projects || template.projects.length === 0) {
				return true;
			}
			// 如果 projects 数组包含当前项目，则模板可被该项目使用
			return template.projects.includes(this.currentProject);
		});
	}
}
```

## 实现要点

### 最小化代码改动

1. **数据结构兼容**：通过 `normalizeBatchTemplate` 函数确保旧版本模板自动获得 `projects: []` 的默认值
2. **UI 渐进增强**：仅在编辑模态框中添加项目选择，不影响现有模板的使用
3. **过滤逻辑**：在选择模态框中进行过滤，不影响数据存储

### 用户体验设计

1. **输入方式**：使用多行文本框，每行一个项目名称，简单直观
2. **提示信息**：明确说明留空表示可被所有项目使用
3. **视觉反馈**：在模板列表中显示项目标签，让用户一目了然

### 数据迁移

1. **无需迁移**：旧模板通过 `normalizeBatchTemplate` 函数自动兼容
2. **向后兼容**：新版本插件可以正常读取旧版本的模板配置
