# 批量创建任务时，给不同层级的任务指定不同的模板

## 背景

当前批量任务模板（`BatchTaskTemplate`）只有一个 `taskType` 字段，所有层级的任务都使用同一种任务类型。实际使用中，不同层级的任务往往需要不同的模板结构——例如：一级任务用"计划"类型、二级任务用"普通"类型、三级任务用"主题"类型。

## 当前实现分析

### 数据结构

```typescript
// batch-task-template.ts
interface BatchTaskTemplate {
	id: string;
	name: string;
	taskType: BatchTaskType; // ← 单一类型，所有层级共用
	listContent: string;
}
```

### 执行流程

`executeBatchCreate()` 中，所有任务统一使用 `template.taskType`：

```typescript
// iotoTasksCenterView.ts - executeBatchCreate()
const result = await createTaskFile({
	type: template.taskType, // ← 所有层级用同一个类型
	templateConfig: this.getTaskTemplateConfig(template.taskType),
	// ...
});

// 父子关系也使用同一类型
const parentTitle = buildBatchTaskTitleForUpTask(
	projectName,
	template.taskType, // ← 父任务也用同一类型
	parentFullName,
);
```

---

## 方案设计

### 核心思路

将 `BatchTaskTemplate` 的 `taskType: BatchTaskType` 替换为 `levelTaskTypes: BatchTaskType[]`，支持最多 3 个层级分别指定任务类型。超出 3 级的部分统一使用 `'normal'`。

### 1. 数据结构变更

**文件：** `src/tasks-center/batch-task-template.ts`

```typescript
// 最大支持的层级数
export const MAX_LEVEL_TASK_TYPES = 3;

// 默认层级任务类型（所有层级都是普通任务）
export const DEFAULT_LEVEL_TASK_TYPES: readonly BatchTaskType[] = [
	'normal',
	'normal',
	'normal',
] as const;

export interface BatchTaskTemplate {
	id: string;
	name: string;
	levelTaskTypes: BatchTaskType[]; // ← 替换原 taskType
	listContent: string;
}
```

**`levelTaskTypes` 数组规则：**

- 数组索引 0 = 第 1 级（顶级）任务类型
- 数组索引 1 = 第 2 级任务类型
- 数组索引 2 = 第 3 级任务类型
- 数组长度不超过 3
- 如果数组为空或长度不足，缺失的层级默认使用 `'normal'`

### 2. 新增辅助函数

```typescript
// 根据层级解析实际的任务类型
export function resolveTaskTypeForLevel(
	levelTaskTypes: BatchTaskType[],
	level: number,
): BatchTaskType {
	if (level < 0) return 'normal';
	if (level < levelTaskTypes.length) {
		return levelTaskTypes[level];
	}
	return 'normal'; // 超出配置范围的层级，默认普通任务
}

// 生成层级类型预览文本（用于 UI 显示）
export function formatLevelTaskTypes(levelTaskTypes: BatchTaskType[]): string {
	return levelTaskTypes
		.map((type, i) => `L${i + 1}: ${getBatchTaskTypeLabel(type)}`)
		.join(', ');
}
```

### 3. 向后兼容：normalizeBatchTemplate 升级

```typescript
export function normalizeBatchTemplate(
	template: Partial<BatchTaskTemplate>,
): BatchTaskTemplate {
	const id = template.id ?? createBatchTemplateId();
	const name = (template.name ?? '').trim();
	const listContent = template.listContent ?? '';

	// 兼容旧版 taskType 字段
	let levelTaskTypes: BatchTaskType[];
	if (template.levelTaskTypes && template.levelTaskTypes.length > 0) {
		levelTaskTypes = template.levelTaskTypes.map((t) =>
			isBatchTaskType(t) ? t : 'normal',
		);
	} else if (template.taskType && isBatchTaskType(template.taskType)) {
		// 旧版：单一 taskType → 所有层级使用同一类型
		levelTaskTypes = [
			template.taskType,
			template.taskType,
			template.taskType,
		];
	} else {
		levelTaskTypes = ['normal', 'normal', 'normal'];
	}

	// 确保长度不超过 MAX_LEVEL_TASK_TYPES
	levelTaskTypes = levelTaskTypes.slice(0, MAX_LEVEL_TASK_TYPES);

	return { id, name, levelTaskTypes, listContent };
}
```

### 4. 执行层变更

**文件：** `src/views/iotoTasksCenterView.ts`

`executeBatchCreate()` 中，每个任务根据自身 `level` 解析类型：

```typescript
private async executeBatchCreate(
    template: BatchTaskTemplate,
    prefix: string,
    items: BatchTaskItem[],
): Promise<void> {
    // ...

    for (const item of items) {
        const fullName = applyPrefix(item.name, prefix);
        // ← 关键变更：根据层级解析任务类型
        const taskType = resolveTaskTypeForLevel(
            template.levelTaskTypes,
            item.level,
        );
        try {
            const result = await createTaskFile({
                // ...
                type: taskType,
                templateConfig: this.getTaskTemplateConfig(taskType),
                // ...
            });
            createdFiles.push({ file: result.file, item });
            // ...
        }
    }

    // 父子关系链接也需要使用正确的类型
    for (const { file, item } of createdFiles) {
        if (item.parentIndex === null) continue;
        const parentEntry = createdFiles[item.parentIndex];
        if (!parentEntry) continue;

        // 父任务的类型 = 父任务自身层级对应的任务类型
        const parentTaskType = resolveTaskTypeForLevel(
            template.levelTaskTypes,
            parentEntry.item.level,
        );
        const parentFullName = applyPrefix(
            parentEntry.item.name,
            prefix,
        );
        const parentTitle = buildBatchTaskTitleForUpTask(
            projectName,
            parentTaskType,        // ← 使用父任务的类型
            parentFullName,
        );
        await assignUpTaskToFile(this.app, file, parentTitle);
    }
    // ...
}
```

编辑弹窗逻辑：

- 解析 `listContent`，检测实际使用的最大层级数
- 只显示到实际最大层级的类型选择器（最多 3 个）
- 如果列表只有 1 级，只显示 1 个选择器
- 如果列表有 2 级，显示 2 个选择器
- 如果列表有 3 级或以上，显示 3 个选择器
- 第 4 级及之后锁定为"普通任务"，有提示说明
