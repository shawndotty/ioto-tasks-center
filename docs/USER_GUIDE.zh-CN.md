# IOTO Tasks Center 使用指南

English: [USER_GUIDE.md](USER_GUIDE.md)

返回首页： [README.zh-CN.md](../README.zh-CN.md)

IOTO Tasks Center 是一个面向 Obsidian 的项目化任务管理插件。它把你的任务继续保留为 Vault 中的普通 Markdown 笔记，同时提供两个专用视图：

- **任务中心**：面向每日任务处理
- **项目中心**：面向项目元数据维护

你可以在不放弃 Obsidian 原生文件结构、wikilink 与 frontmatter 的前提下，获得更适合项目推进的任务管理体验。

## 插件概述

插件会把：

- 任务根目录下的**一级文件夹**视为项目
- 每个项目目录下的**一级 Markdown 文件**视为任务文件

默认任务根目录：`3-任务`

也就是说，IOTO Tasks Center 并不会维护一个独立数据库，而是直接读取和写入你的 Vault 文件。你仍然可以继续使用：

- Markdown
- wikilink
- frontmatter
- 模板系统
- Obsidian 原生预览与编辑能力

## 2.1.1 版本重点更新

`2.1.1` 主要围绕任务列表交互进行了增强：

- 任务列表支持可选显示子任务数量 Badge
- 子任务与出链 Badge 支持 **多彩配色 / 单色背景** 两种模式

## 主要功能

### 任务中心

- 左侧项目列表，显示每个项目的未完成任务数
- 根据视图宽度自动切换普通布局与紧凑布局
- 右侧任务列表支持筛选、搜索、排序、分组
- 右侧固定任务预览 pane
- 常见刷新与操作场景下保留滚动位置

### 项目中心

- 独立的项目表格管理视图
- 支持搜索、刷新、新建项目
- 支持维护项目分类、开始日期、截止日期、是否存档
- 支持按列排序
- 项目“存档”会与任务中心中的“隐藏项目”行为联动

### 任务创建

- 支持四种任务类型：
  - 普通任务
  - 日期任务
  - 主题任务
  - 计划任务
- 支持为每种任务类型分别配置模板
- 模板来源支持：
  - 模板文件
  - 直接输入模板内容
- 日期任务支持自定义日期格式
- 非日期任务的创建弹窗支持直接设置：
  - 任务名称
  - 优先级
  - 是否核心任务

### 子任务与层级

- 可通过拖拽把任务设置为另一个任务的子任务，写入 `UpTask`
- 可拖拽到“移除父任务”区域清除 `UpTask`
- 子任务支持层级缩进显示
- 父任务支持折叠/展开子任务树
- 右键菜单支持直接创建子任务
- 支持把编辑器中选中的文本转换为新的子任务笔记
- 支持可选显示“直接子任务数量 Badge”，并在悬浮时弹出子任务列表

### 任务元数据与状态

- 任务优先级写入 frontmatter，可用于排序和分组
- 核心任务支持星标显示
- 任务状态根据笔记中的 checklist 自动计算
- 当前支持的状态包括：
  - 待开始
  - 进行中
  - 已完成
  - 无任务
- `待开始` / `进行中` 状态 Badge 悬浮时可显示未完成 checklist
- 点击 checklist 条目时可打开原任务笔记并选中对应行

### 出链洞察

- 可选显示任务到以下目录的出链数量：
  - 输入笔记根目录
  - 输出笔记根目录
  - 成果笔记根目录
- 悬浮 Badge 可弹出对应链接列表
- 点击条目可在右侧预览 pane 中打开对应笔记
- 在出链 popover 中按住 `Cmd` / `Ctrl` 悬浮，可使用 Obsidian 原生链接预览

### 搜索、筛选与组织

- 任务筛选标签：
  - 核心任务
  - 今天
  - 未完成
  - 已完成
  - 全部
- 搜索使用独立 popover，支持聚焦、清空和键盘操作
- 任务排序支持：
  - 创建时间
  - 更新时间
  - 文件名
  - 优先级
- 任务分组支持：
  - 不分组
  - 按状态分组
  - 按优先级分组
- 分组区域支持折叠与展开

### 任务右键菜单

在任务列表中右键某个任务，可以执行以下操作：

- 标记或取消标记为核心任务
- 添加子任务
- 设置或清除优先级
- 删除任务

## 文件组织方式

插件不会改变你的 Vault 结构，也不会创建外部数据库，而是直接基于文件系统工作。

- 任务根目录：Vault 相对路径，默认 `3-任务`
- 项目：任务根目录下的一级文件夹
- 任务文件：项目目录下的一级 Markdown 文件

示例：

```text
3-任务/
  项目 Alpha/
    Kickoff.md
    Alpha-2026-06-15.md
  项目 Beta/
    Release review.md
```

注意：任务列表只扫描项目目录下的第一层 Markdown 文件。子任务层级依赖 frontmatter 中的 `UpTask`，而不是依赖子目录结构。

## Frontmatter 字段

插件当前会读写以下字段：

| 字段 | 类型 | 用途 |
| --- | --- | --- |
| `Project` | YAML 列表 | 记录所属项目名称 |
| `UpTask` | YAML 列表 | 记录父任务 wikilink，例如 `[[父任务]]` |
| `Priority` | 数字标量 | 记录任务优先级 |
| `Subject` | YAML 列表 | 主题任务使用 |
| `Plan` | YAML 列表 | 计划任务使用 |
| `Starred` | 布尔值 | 标记是否为核心任务 |

## 命令

当前插件提供这些命令：

- **Open tasks center view**
- **Open project center view**
- **Convert selected text to subtask**

其中“选中文本转子任务”命令具备这些行为：

- 仅在当前活动文件位于任务根目录下时可用
- 新子任务会创建在与当前任务相同的项目目录中
- 新子任务会继承当前任务的 `Project`
- 自动写入指向当前任务的 `UpTask`
- 会把原选中文本替换成指向新子任务的 wikilink

## 设置项说明

### 基础设置

- **任务根目录**
  - 用于发现项目的 Vault 相对路径
- **输入 / 输出 / 成果笔记根目录**
  - 用于计算任务出链数量和分类
- **打开任务中心**
  - 在设置页中直接打开任务中心
- **打开项目中心**
  - 在设置页中直接打开项目中心
- **日期任务日期格式**
  - 默认值：`YYYY-MM-DD`

### 任务列表展示

- **项目列表排序**
  - 按未完成任务数
  - 按名称
- **任务排序方式**
  - 创建时间降序 / 升序
  - 更新时间降序 / 升序
  - 文件名升序 / 降序
  - 优先级降序 / 升序
- **任务分组方式**
  - 不分组
  - 按状态分组
  - 按优先级分组
- **显示任务优先级**
- **显示子任务数量**

### 出链 Badge

- **显示任务出链计数**
- **显示输入出链数量**
- **显示输出出链数量**
- **显示成果出链数量**
- **任务列表链接 Badge 背景色**
  - `多彩配色`（默认）
  - `单色背景`

### 任务类型

可以分别启用或禁用以下创建类型：

- 普通任务
- 日期任务
- 主题任务
- 计划任务

至少需要保留一种任务类型启用。

### 任务模板

每一种任务类型都可以独立配置模板：

- 来源模式：`模板文件` 或 `直接输入模板内容`
- 模板文件路径
- 模板内容

Templater 是可选集成，不是必需依赖。如果你希望模板创建时交给 Templater 处理，可在这里配置对应模板文件路径。

## 快速开始

### 1. 准备目录结构

例如先在 Vault 中创建：

```text
3-任务/
  我的项目/
```

### 2. 打开任务中心

可通过以下任一方式打开：

- 命令面板
- 插件设置页中的入口按钮

### 3. 创建第一个任务

- 选择一个项目
- 点击添加任务
- 选择任务类型
- 如果不是日期任务，可在弹窗中填写：
  - 名称
  - 优先级
  - 是否核心任务

### 4. 建立任务层级

- 把一个任务拖到另一个任务上，建立父子关系
- 或者右键任务，直接选择“添加子任务”

### 5. 跟踪推进情况

- 通过状态 Badge 查看 checklist 进度
- 悬浮状态 Badge 查看未完成 checklist
- 悬浮出链 Badge 或子任务 Badge 快速跳转相关笔记

## 使用示例

### 示例 1：每日核心任务检查

- 打开 **任务中心**
- 切换到 **核心任务** 标签
- 按优先级排序
- 悬浮 `进行中` 状态 Badge，查看未完成 checklist
- 如果需要处理细节，直接点击 popover 中的 checklist 条目进入笔记

### 示例 2：构建任务树

- 创建一个父任务，例如 `发布计划`
- 右键该任务并选择“添加子任务”
- 继续创建 `撰写文案`、`准备素材`、`测试发布` 等子任务
- 悬浮子任务数量 Badge，可快速查看并打开直接子任务

### 示例 3：查看任务关联笔记

- 配置输入 / 输出 / 成果笔记根目录
- 开启出链计数
- 悬浮出链 Badge 查看当前任务链接到了哪些相关笔记
- 点击条目，在右侧预览 pane 打开对应笔记

## 安装

### 方式一：通过社区插件安装

如果该插件已进入 Obsidian 社区插件目录，可按以下步骤安装：

1. 打开 **设置 → 社区插件**
2. 如有需要，先关闭安全模式
3. 搜索 `IOTO Tasks Center`
4. 安装并启用

### 方式二：手动安装

1. 从 [GitHub Releases](https://github.com/shawndotty/ioto-tasks-center/releases) 下载最新发布包
2. 将以下文件复制到：

   ```text
   <你的 Vault>/.obsidian/plugins/ioto-tasks-center/
   ```

3. 至少需要复制：
   - `main.js`
   - `manifest.json`
   - `styles.css`
4. 重载 Obsidian
5. 在 **设置 → 社区插件** 中启用插件

## 国内环境使用说明

- 由于发布资源托管在 GitHub Releases，部分网络环境下下载速度可能较慢或无法直接访问。
- 如果无法直接下载发布包，可以：
  - 在本地克隆仓库并执行 `npm run build`
  - 手动复制 `main.js`、`manifest.json`、`styles.css` 到插件目录
- 如果你通过同步工具在多设备间共享 Vault，建议先在桌面端完成模板、路径与目录结构配置，再同步到移动端使用。

## 兼容性说明

- 插件不是桌面专用，设计目标是桌面端与移动端都可使用。
- 但以下交互在桌面端体验会更好：
  - 拖拽建立任务层级
  - 悬浮预览
  - 基于 hover 的 popover
- 插件完全基于本地 Markdown 文件工作，不依赖远程服务。
- Templater 为可选集成，不安装也可以正常使用。

## 常见问题排查

### 任务中心打开后是空的

请检查：

- 任务根目录是否配置正确
- 项目文件夹是否直接位于任务根目录下
- 任务文件是否直接位于项目目录下，而不是更深的子目录中

### 某个任务没有显示在列表中

常见原因包括：

- 文件不在配置的任务根目录下
- 文件位于未被扫描的嵌套子目录里
- 当前筛选标签把它过滤掉了
- 当前搜索关键词把它过滤掉了

### 拖拽设置父任务失败

请检查是否：

- 把任务拖到了它自己身上
- 把任务拖到了自己的后代任务上
- 目标任务当前不可用或不可见

### 出链 Badge 没有显示

请检查：

- 是否开启了 **显示任务出链计数**
- 输入 / 输出 / 成果笔记根目录是否配置正确
- 当前任务是否真的链接到了这些根目录下的笔记
- 计数是否大于 0，因为 0 不会显示 Badge

### 状态 popover 里没有内容

如果任务笔记中没有未完成 checklist，popover 为空属于正常现象。

### 子任务数量 Badge 没有显示

请检查：

- 是否开启了 **显示子任务数量**
- 当前任务是否真的有直接子任务
- 子任务是否已正确写入 `UpTask`
