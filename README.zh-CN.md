# IOTO Tasks Center

English: [README.md](README.md)

在 Obsidian 中按“项目”管理 Markdown 任务文件。插件提供独立的“任务中心”视图，支持项目导航、任务列表筛选/搜索/排序/分组，以及轻量的任务元数据（UpTask / Priority / Project）。

## 概述

IOTO Tasks Center 会把你配置的“任务根目录”下的**一级文件夹**当作项目（Project），并把每个项目里的 Markdown 文件当作任务文件（Task files）进行管理与展示。

默认任务根目录：`3-任务`

## 主要功能

- 项目列表：展示任务根目录下的项目（文件夹），支持排序与隐藏
- 任务列表：展示当前项目下的任务文件（Markdown 文件）
- 筛选标签：今天 / 未完成 / 已完成 / 全部
- 搜索：按任务文件名搜索（仅作用于当前项目）
- 排序：按创建/更新时间、名称、优先级排序
- 分组：不分组 / 按状态分组 / 按优先级分组（支持折叠/展开）
- 状态统计：扫描文件内 checkbox 任务，显示完成/总数摘要
- 优先级：为任务文件设置/清除 `Priority`（0–9），并用于排序/分组
- 父子任务（UpTask）：拖拽任务到另一个任务上建立父子关系（写入 `UpTask`）；拖拽到“移除父任务”区域可清除父任务
- 悬停预览：鼠标悬停任务行快速预览任务内容
- 选中文本转子任务：把当前任务文件中的选中文本创建为新子任务，并将选中文本替换为 wikilink
- 模板创建任务：创建任务文件时可使用模板（Templater 模板文件 / 直接输入模板内容），并支持按任务类型分别配置

## 文件组织方式

- 任务根目录（vault 相对路径）：默认为 `3-任务`
- 项目：任务根目录下的一级文件夹
- 任务文件：项目目录下的一级 Markdown 文件（任务列表只读取这一层，不递归子目录）

示例结构：

```text
3-任务/
  项目A/
    需求梳理.md
    项目A-2026-06-04.md
  项目B/
    发布复盘.md
```

## Frontmatter 字段

插件会读写这些 frontmatter 字段：

- `Project`：list，项目名列表（会写成 YAML 列表）
- `UpTask`：list，父任务 wikilink，例如 `[[父任务]]`
- `Priority`：标量数字（0–9）
- `Subject`：list，“主题任务”类型会写入
- `Plan`：list，“计划任务”类型会写入

## 命令

- 打开任务中心视图
- 选中文本转子任务
  - 仅当当前文件位于任务根目录下时可用。
  - 子任务会创建在与当前任务相同的项目与目录下。
  - 新子任务会继承 `Project`，并写入指向当前文件的 `UpTask`。

## 设置项

- 任务根目录：用于发现项目
- 视图入口：从设置页直接打开任务中心视图
- 任务模板
  - 按任务类型分别配置：日期 / 计划 / 主题 / 普通
  - 模板来源：模板文件（可配合 Templater）或直接输入模板内容
- 日期任务日期格式：控制日期任务文件名中的日期片段（Moment/Day.js 风格）
- 项目排序：按未完成数量（默认）或按名称
- 隐藏项目：从项目列表隐藏选中的项目
- 任务列表展示
  - 排序方式、分组方式、是否显示优先级

## 安装

- 社区插件：（如已发布）在 **设置 → 第三方插件/社区插件** 中安装。
- 手动安装：
  - 将 `main.js`、`manifest.json`、`styles.css`（如有）复制到：
    - `<Vault>/.obsidian/plugins/ioto-tasks-center/`
  - 重载 Obsidian，并在 **设置 → 第三方插件/社区插件** 中启用插件。
