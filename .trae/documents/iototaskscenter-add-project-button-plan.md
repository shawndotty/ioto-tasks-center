## Summary
- 在左侧“项目列表”标题右侧新增一个弱化样式的添加项目按钮。
- 该按钮不显示文字，仅显示加号图标，用于创建新项目目录。
- 用户点击后弹出输入窗口，输入项目名称后在 `3-任务/` 下创建对应项目文件夹。
- 创建成功后立即刷新左侧项目列表，并选中新建项目。

## Current State Analysis

  - 项目列表的加载、排序、隐藏过滤、选中切换、刷新逻辑都已经集中在该文件中，新增“创建项目”入口也应继续落在这里统一管理。

  - 当前已经有一个通用的输入型 modal，支持标题、placeholder、确认/取消、回车提交。
  - 该 modal 虽然命名偏任务，但实际能力可以复用于项目名称输入，或者在本次实现中抽象/重命名为更通用的名称输入弹窗。
- `styles.css`
  - 已存在通用的 `.ioto-tasks-center__section-header`、`.ioto-tasks-center__section-actions` 以及按钮相关样式。

- `src/tasks-center/types.ts`
  - 已定义任务根目录常量 `TASKS_ROOT_PATH = '3-任务'`，可直接复用，避免手写路径字符串。
- 当前仓库中尚未发现专门的“项目创建”模块或相关测试文件，说明这一能力需要新增最小范围的创建逻辑与测试覆盖。

## Proposed Changes
### 1. `src/views/iotoTasksCenterView.ts`
- 将左侧项目栏标题从纯文本改为“标题 + 操作区”结构：
  - 左侧保留 `项目列表`
  - 右侧新增 icon-only 的“添加项目”按钮
- 新增项目创建相关状态与方法，建议与现有任务创建保持同一风格：
  - `isCreatingProject`
  - `canCreateProject()`
  - `getAddProjectButtonLabel()`
  - `handleCreateProject()`
- 点击按钮后的流程：
  - 若当前正在加载项目列表，则按钮禁用
  - 打开输入 modal，让用户输入项目名称
  - 若取消或输入为空，则中止
  - 调用项目创建逻辑在 `3-任务/<项目名>` 下创建文件夹
  - 创建成功后刷新项目列表，并自动选中新创建的项目
  - 若同名项目已存在，则不重复创建，直接选中该项目并给出提示
- 保持与现有刷新机制兼容，不绕开 `refreshFromVaultChange()` / `selectProject()` 现有流程。

### 2. 新增或抽离项目创建逻辑模块
- 建议新增一个轻量模块，例如 `src/tasks-center/project-creation.ts`，避免把文件夹创建、名称清洗、重名处理全部堆进视图层。
- 该模块职责建议包括：
  - `normalizeProjectName(input)`
    - 去除首尾空白
    - 折叠连续空白
    - 替换路径非法字符
    - 空结果返回 `null`
  - `resolveProjectTargetPath(projectName)`
    - 返回 `3-任务/<项目名>`
  - `createProjectFolder(app, projectName)`
    - 若 `3-任务` 根目录不存在，则抛错或提示无法创建
    - 若目标目录已存在，则返回“未新建但可直接选中”的结果
    - 若不存在，则创建文件夹并返回创建成功结果
- 这样能让后续测试聚焦纯逻辑和最小行为，而不是直接耦合视图 DOM。

### 3. `src/ui/taskNameModal.ts` 或新的通用名称弹窗
- 当前 modal 文案写死为“请输入新任务文件的名称。”，不完全适合项目创建。
- 两种实现方向中，本次计划采用更稳妥的通用化方案：
  - 将现有 modal 调整为更通用的名称输入弹窗，比如支持传入描述文案与确认按钮文本
  - 任务创建继续用它，项目创建也复用它
- 若不想大改现有任务创建调用，也可新建一个更通用的 `NameInputModal` 并逐步替换，但本次只需保证项目创建的文案准确。

### 4. `styles.css`
- 为左侧新增项目按钮补充弱化图标样式，例如：
  - `.ioto-tasks-center__icon-button`
  - hover / disabled / focus-visible 状态
- 样式目标：
  - 尺寸小于右侧文字按钮
  - 默认低对比度、视觉弱化
  - 用 `+` 作为按钮文本或图标内容，不显示“添加项目”文字
  - 通过 `title` / `aria-label` 保留可访问性
- 保持窄屏下左侧标题区不挤压标题文字。

### 5. 测试
- 新增测试文件，例如 `tests/project-creation.test.mjs`
- 聚焦纯逻辑测试，不为 Obsidian 运行时文件夹创建写高噪声 mock：
  - `normalizeProjectName()` 对空白、非法字符、连续空格的处理
  - `resolveProjectTargetPath()` 返回 `3-任务/<项目名>`
- 如项目创建模块返回可判定的状态值，也可补充结果结构测试。
- 现有 `tests/project-sort.test.mjs`、`tests/task-creation.test.mjs` 保持不动，避免把无关逻辑混在一起。

## Assumptions & Decisions
- 新按钮放在左侧项目列表标题右侧，与右侧任务列表标题区的结构保持一致。
- 按钮采用弱化 icon-only 形式，只显示 `+`，不显示文字。
- 点击后使用弹窗输入项目名称，而不是内联输入框或右键菜单。
- 项目创建目标固定为 `3-任务/<项目名>/` 一级目录，不创建默认任务文件。
- 若项目已存在，则不报阻断错误；默认行为定为提示并直接选中已有项目。
- 若 `3-任务` 根目录不存在，则不自动创建根目录；沿用当前插件对根目录缺失的处理方式，提示用户先创建根目录。
- 本次只实现“添加项目目录”，不顺带扩展项目重命名、删除、排序规则调整或创建后自动生成示例任务。
