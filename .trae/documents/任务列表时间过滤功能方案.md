# 任务列表时间过滤功能方案

## 概述

在任务列表展示菜单（`showTaskPresentationMenu`）中新增「Filter」组，用户可按创建时间（ctime）或更新时间（mtime）过滤任务，可选时间段：最近一周、最近两周、最近一月。

## 当前状态分析

- `TaskFileEntry` 已有 `ctime` 和 `mtime` 字段（Unix 毫秒时间戳），来自 Obsidian 的 `TFile.stat`（[types.ts](file:///Users/johnny/Documents/Sync/IOTO-Plugins/.obsidian/plugins/ioto-tasks-center/src/tasks-center/types.ts#L44-L57)）
- 现有过滤管道：tab 过滤（`getTasksForActiveTab`）→ 搜索过滤（`getVisibleTasks`）→ 排序/分组
- 设置系统模式：类型定义 → 接口字段 → 默认值 → main.ts 回调 → 视图访问器
- 菜单模式：`formatMenuOptionTitle(category, label, active)` 构建标题，`menu.addSeparator()` 分隔组
- `showTaskPriority` 设置无独立 settings UI，仅通过菜单切换——时间过滤跟随相同策略

### 变更 4：新建 task-time-filter.ts — 时间过滤纯函数

**新建文件**：`src/views/tasks-center/task-time-filter.ts`

```typescript
import type { TaskFileEntry } from '../../tasks-center/types';
import type { TaskListTimeFilter } from '../../settings';

const DAY_MS = 24 * 60 * 60 * 1000;
const WEEK_MS = 7 * DAY_MS;
const TWO_WEEKS_MS = 14 * DAY_MS;
const MONTH_MS = 30 * DAY_MS;

function getThreshold(filter: TaskListTimeFilter): number {
    const now = Date.now();
    switch (filter) {
        case 'created-week':
        case 'updated-week':
            return now - WEEK_MS;
        case 'created-two-weeks':
        case 'updated-two-weeks':
            return now - TWO_WEEKS_MS;
        case 'created-month':
        case 'updated-month':
            return now - MONTH_MS;
        default:
            return 0;
    }
}

export function filterTasksByTime(
    tasks: TaskFileEntry[],
    filter: TaskListTimeFilter,
): TaskFileEntry[] {
    if (filter === 'none') {
        return tasks;
    }

    const threshold = getThreshold(filter);
    const useCTime = filter.startsWith('created');

    return tasks.filter((task) => {
        const time = useCTime ? task.ctime : task.mtime;
        return time >= threshold;
    });
}
```

**预期效果**：新文件约 40 行。

## 架构决策

1. **过滤器放在搜索之后**：时间过滤在 tab 过滤和搜索过滤之后执行，作为补充过滤层，不影响搜索和 tab 的行为。
2. **纯函数实现**：`filterTasksByTime` 是纯函数，无副作用，易于测试。
3. **菜单优先**：新增设置不添加独立的 settings UI，仅通过任务列表展示菜单操作——与 `showTaskPriority` 一致。
4. **时间阈值用毫秒**：与 `TaskFileEntry.ctime/mtime` 单位统一，避免转换。
5. **`none` 为默认值**：默认不过滤，保持对现有行为的完全兼容。
6. **不修改 `getTasksForActiveTab`**：时间过滤不侵入现有的 tab 过滤逻辑，保持关注点分离。
7. **不持久化到视图 State**：时间过滤设置属于全局设置（跨视图一致），而非视图级别状态，通过 `settings` 持久化。
