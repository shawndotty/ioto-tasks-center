# 批量创建任务增加任务名后缀方案

## 概述

在批量创建任务的流程中，当前仅支持设置任务名「前缀」。本次扩展为同时支持「前缀」和「后缀」：把现有的 `BatchPrefixModal` 升级为 `BatchNameAffixModal`，返回 `{ prefix, suffix }`，并在预览/创建逻辑中统一通过 `applyAffix` 应用前后缀。

本方案基于 `plans/批量创建任务时，增加任务名后缀设置的方案.md`，结合实际代码（行号、签名、i18n 三语、测试约定）做了落地细化与一致性修正。

### 关键观察

1. **`applyPrefix` 在 `executeBatchCreate` 中被调用 2 处**：一次用于子任务文件名（第 933 行），一次用于父任务标题构建（第 974 行）。两处都必须改为支持后缀，否则父子关联会因名称不一致而失败。
2. **`formatBatchItemsForPreview` 签名是位置参数** `(items, prefix, levelTaskTypes)`，新增 suffix 需要插入到中间位置，调用方（确认弹窗）需同步更新。
3. **`modal.batchConfirm.summary` 现为 3 占位符**（模板/前缀/项目），方案文档要求改为 4 占位符（加入后缀）。这是破坏性 i18n 变更，必须三语同步。
4. **`BatchPrefixModal` 取消语义用 `null`**（空串表示「无前缀」继续创建）。新的 `BatchNameAffixModal` 应沿用：返回 `null`=取消，返回 `{prefix, suffix}`=继续（两者均可为空串）。
5. **`BatchPrefixModal` 是否还有其他调用方**：需确认仅 `triggerBatchCreateFromTemplate` 使用（见验证步骤）。

## 拟定变更

### 变更 1：数据层——新增 `applySuffix` 与 `applyAffix`

**文件**：`src/tasks-center/batch-task-template.ts`

在 `applyPrefix`（第 148-150 行）之后新增两个导出函数：

```typescript
export function applySuffix(name: string, suffix: string): string {
	return `${name ?? ''}${suffix ?? ''}`;
}

export function applyAffix(
	name: string,
	prefix: string,
	suffix: string,
): string {
	return `${prefix ?? ''}${name ?? ''}${suffix ?? ''}`;
}
```

- **保留** `applyPrefix` 不删除（避免破坏潜在的其他引用与既有测试），`applyAffix` 作为前后缀统一入口。
- **为什么**：`executeBatchCreate` 与 `formatBatchItemsForPreview` 都需要同时应用前后缀，统一函数避免逻辑分散。

### 变更 2：预览函数支持后缀

**文件**：`src/tasks-center/batch-task-template.ts`

修改 `formatBatchItemsForPreview`（第 190-199 行）签名，在 `prefix` 之后插入 `suffix`：

```typescript
export function formatBatchItemsForPreview(
	items: BatchTaskItem[],
	prefix: string,
	suffix: string = '',
	levelTaskTypes: BatchTaskType[] = [...DEFAULT_LEVEL_TASK_TYPES],
): Array<{ indent: number; text: string; taskType: BatchTaskType }> {
	return items.map((item) => ({
		indent: item.level,
		text: applyAffix(item.name, prefix, suffix),
		taskType: resolveTaskTypeForLevel(levelTaskTypes, item.level),
	}));
}
```

- `suffix` 设默认值 `''` 以向后兼容现有测试（现有用例仅传 3 个位置参数）。
- **注意位置参数顺序**：`suffix` 插在 `prefix` 与 `levelTaskTypes` 之间，调用方需同步。

## 假设与决策

1. **重命名而非新建**：将 `BatchPrefixModal` 重命名为 `BatchNameAffixModal`，不保留旧别名，同步更新唯一调用方。符合「避免 backwards-compatibility hacks」原则。
2. **保留 `applyPrefix` 导出**：不删除既有函数（其测试仍在），新增 `applySuffix`/`applyAffix`。`executeBatchCreate` 与 `formatBatchItemsForPreview` 内部改用 `applyAffix`。
3. **取消语义**：`BatchNameAffixModal` 返回 `null`=取消（与原 `BatchPrefixModal` 一致）；返回 `{prefix, suffix}`=继续，两者均可为空串。
4. **`suffix` 参数位置**：在 `formatBatchItemsForPreview` 中插在 `prefix` 与 `levelTaskTypes` 之间，并设默认值 `''` 以兼容既有测试。
5. **i18n 清理**：删除不再使用的 `modal.batchPrefix.*` 三键（三语），避免孤儿键。
6. **`executeBatchCreate` 两处 `applyPrefix` 都要改**：子任务文件名与父任务标题必须同时用 `applyAffix`，否则父子关联会因名称不匹配而失败。
7. **父子标题一致性**：`buildBatchTaskTitleForUpTask` 接收的是已拼好前后缀的 `parentFullName`，无需修改该函数本身。
